# Process TEA Tool

Build a process flow diagram (PFD) for electrochemistry / plastic / biomass / hydrogen / CO2RR processes and generate a paper-quality **Techno-Economic Analysis** (CAPEX / OPEX / Revenue / MSP / sensitivity), exportable to xlsx in the same layout as the reference paper TEA Summary (`260402 TEA summary.xlsx`).

---

## 🟢 빠른 시작 — 코딩 필요 없음 / Quick start (no coding)

**한국어:** 상위 폴더의 **`TEA_보고서_생성.bat`** 파일을 더블클릭하세요.
모든 논문의 경제성 분석 보고서(PFD·스트림표·CAPEX/OPEX·MSP·민감도·참고문헌)를
자동 생성하고, 브라우저에서 목록 페이지(`dossier/index.html`)를 엽니다.
보고서 한 장에 공정도부터 출처까지 전부 들어 있고, 설치/서버 불필요합니다.

**English:** Double-click **`TEA_보고서_생성.bat`** in the parent folder. It
generates a self-contained HTML "dossier" for every paper and opens the index
in your browser. Each dossier has the PFD, stream table, CAPEX/OPEX, MSP,
sensitivity, and sourced references — one file, no server, no install.

Prefer the command line?

```bash
python generate_dossiers.py          # build dossier/*.html + index.html
# then open tea_tool/dossier/index.html in any browser
```

Want the interactive explorer (sliders, live edits)? Use the Streamlit app
(`run.bat`) — but for "just show me the numbers," the dossier above is enough.

> Each dossier loads Mermaid (PFD) and Chart.js (sensitivity) from a CDN on
> first open, so keep internet on the first time; tables/text work offline.

## AI-assisted workflow (experiment → process → TEA → scale-up)

The tool now supports a **lab-data-first** workflow where you describe an
experiment once in YAML and the tool / Claude Code build the rest:

```
experiments/<slug>.yaml      ←  filled via the 🧪 Lab Data tab (or by hand)
        │
        │  build_process_from_experiment()
        ▼
processes/from_experiment_<slug>.py   (optional, Claude Code refines this)
        │
        │  run_tea()
        ▼
TEA tables + xlsx export      ← TEA / Export tabs
        │
        │  build_scaleup_report()
        ▼
📈 Scale-up tab: 5-stage ladder (lab → bench → pilot → demo → commercial)
        + reaction-class scale-up risks
        + EHS / safety notes
        + recommendation against an MSP target
```

The minimum YAML keys to bootstrap the pipeline are:

```yaml
meta:          { name, slug }
chemistry:     { reaction_type, target_products }
feedstock:
  primary:     { name, mass_per_batch_g }
results:
  yields:      [{ product, yield_pct }, ...]
```

Everything else is optional but improves the quality of the auto-generated
PFD, capex defaults, and scale-up risk catalog. See
[`experiments/SCHEMA.md`](experiments/SCHEMA.md) for the full spec and
[`experiments/example_lignin_pma_mw.yaml`](experiments/example_lignin_pma_mw.yaml)
for a complete reference.

### Working with Claude Code

After you save an experiment YAML, the **🧪 Lab Data** tab shows an
auto-generated instruction block. Paste it into the Claude Code CLI and the
agent will:

1. Compare 2-3 process design options for the chemistry and pick one with
   a justification paragraph.
2. Write `processes/from_experiment_<slug>.py` with vendor-grade equipment
   costs and cited assumptions.
3. Save a design note at `design_notes/<slug>.md` (option comparison + key
   assumptions + scale-up risks).
4. Run the TEA and report MSP across the scale ladder.

If you just want a smoke-test result without involving the agent, click the
**🚀 Bootstrap design note + xlsx** button on the right of the tab — it
runs `build_process_from_experiment` immediately and drops a starter design
note in `design_notes/<slug>.md`.

## Where do prices come from?

`data/prices.yaml` is the canonical price database.  Each entry carries:
- `price_low` / `price_high` ($/kg)
- `tier` — `paper` (locked-in from a TEA paper), `market` (auto-refresh enabled), `estimate`
- `source` — paper citation or market name
- `lookup` — list of public URLs the user can click to verify the latest price

### Automatic refresh (no API keys needed)

Click **🌐 Refresh prices from web** in the 💰 **Prices** tab.  The tool
pulls live data from these public sources:

| Source                            | What it provides | Frequency | Coverage |
|---|---|---|---|
| **World Bank Pink Sheet** (xlsx)  | Crude oil, Brent, natural gas (US/EU/Japan), coal, naphtha, urea, DAP, KCl, phosphate rock, soybeans, maize, wheat, sugar, rice, cotton, rubber, aluminum, copper, lead, tin, zinc, nickel, iron ore | monthly | 22 commodities |
| **Trading Economics** (HTML)      | Methanol, polyethylene, polypropylene, PVC, styrene, sulfur, urea, DAP, ethanol, propane, naphtha, lithium, coking coal, plus FX-converted base metals | daily | ~30 commodities |
| **Frankfurter (ECB)**             | CNY → USD FX conversion for the Trading-Economics CNY-quoted prices | daily | n/a |

That covers ~50 commodity-grade chemicals, energy carriers, fertilizers,
biomass feedstock proxies and metals — automatically, with timestamps and
provenance recorded in each component's `source` field.

The fetcher module lives in [`tea_engine/fetchers.py`](tea_tool/tea_engine/fetchers.py).
Adding a new auto-fetched source means adding one function that returns
`PriceRecord(component, price_USD_per_kg, unit_original, source, date_str)`
items.

### Manual lookup for specialty chemicals

DMSO, PMA, hydroxylamine and other specialty chemicals have no free public
feed (ECHEMI / Made-in-China are public but Cloudflare-protected against
automated requests).  For these the **🔍 Manual lookup links** section
gives one-click buttons to open ECHEMI / Made-in-China / Alibaba / ICIS
search pages in your browser; copy the latest price and paste into the
editor table, then **💾 Save** to persist to YAML.

### Why aren't ICIS / S&P Global feeds auto-pulled?

Those are subscription-only research products, not commodity exchanges.
The Pink Sheet and Trading Economics feeds we *do* pull are the same
spot-price data that flows into ICIS reports — just sourced upstream of
the analyst markup.

### What's locked vs. live

- `tier: paper` — never overwritten by the auto-refresh.  These are the
  authoritative numbers from the source TEA paper (e.g. Bagemihl uses CO2
  at $0.04/kg, Mosalpuri assumes free waste-NO3⁻ feedstock); changing them
  would invalidate the paper's reproduction.  Toggle **"Lock paper-tier
  prices"** off if you want to overwrite anyway.
- `tier: market` — overwritten on every refresh.
- `tier: estimate` — overwritten on every refresh.

## Time-resolved flows (📅 Time Profile tab)

Inputs to a real plant come in three flavours, and economically they look
nothing alike:

| Mode | What it is | Examples | Where the cost lands |
|---|---|---|---|
| `continuous` *(default)* | Fed every batch / every month | Feedstock, electricity, water | OPEX (feedstock + utility lines) |
| `one_time` | Loaded once at t=0 (initial inventory) | Catholyte, electrolyte, solvent fill | CAPEX-extra (charge × CRF) |
| `periodic` | Fully replaced every N months | MEA, ion-exchange resin, catalyst | OPEX (charge × $/kg ÷ N) |

Equipment items get the same treatment via `lifetime_years` /
`replacement_interval_years` — an electrolyzer stack with a 7 y life inside a
20 y plant is annualized at *its own* CRF rather than the plant CRF, which
correctly captures the recurring stack swap.

Declare these on a `Stream`:
```python
Stream("KHCO3", 0.10, recovery=0.0,
       flow_mode="one_time", initial_charge_kg_per_ton=50.0)

Stream("MEA membrane", 0.0,
       flow_mode="periodic",
       initial_charge_kg_per_ton=100.0,
       replacement_interval_months=60)
```
…and on `Equipment`:
```python
Equipment("GDE cell stack", "Alkaline GDE Electrolyzer",
          base_cost=5_500_000, cap_ref=1.0,
          lifetime_years=7)
```

The new **📅 Time Profile** tab in the app shows:
- Stacked-area: monthly $ spent on each input stream, colour-keyed by mode
  (continuous = blue, one_time = orange, periodic = purple).
- Step line: on-site inventory of one_time / periodic streams (sawtooth on
  replacement, decay-only for one_time).
- Cash-flow bar with cumulative NPV overlay — CAPEX at month 0, OPEX every
  month, replacement spikes on event months.
- Event tables listing every stream and equipment replacement.

PFD nodes (both the graphviz view and the streamlit-flow editor) now carry a
small badge — `♻ cont.`, `◆ initial only (50 kg/ton)`, or `⏳ every 5 y` — so
the time semantics are visible directly on the diagram.

## Interactive Flow Editor

The **🧩 Flow Editor** tab is a ReactFlow-style canvas:

- **Drag** any node to reposition it.
- **Drag from a node's right handle to another node's left handle** to add an edge.
- **Right-click** a node or edge for a context menu (delete, edit label).
- **Click** a node to expose inline number inputs that edit
  - input streams: g/batch, recovery, $/kg
  - output streams: g/batch, $/kg
  - process steps: name, description
- The **➕ Add a node** expander adds new input/output streams or new process
  steps; a placeholder equipment unit is auto-added for new steps so the
  CAPEX section list updates immediately.
- All edits flow into the same `Process` data model that drives the TEA
  calculation, so the tables, plots and xlsx export update on every
  rerun.

The "Streams" and "Equipment" tabs remain available for bulk table editing.

## Validation against the reference paper

`py -3 smoke_test.py` exercises the PET template and prints a side-by-side
comparison with the source xlsx.  All section CAPEX, total CAPEX, annualized
CAPEX, revenue, and 1- and 5-ton OPEX/MSP match to the dollar; the 10-ton
column differs by the value of one extra ton of PET feedstock because the
source workbook accidentally fixed the PET feedstock column at 5-ton level
(this engine scales it linearly, which is the intended physical behavior).

## Adding a new process

Drop a new file in `processes/` that exposes a `build()` returning
`(Process, ComponentDB, TEAInputs)`, then register it in
`processes/__init__.py`.  The builder must:

1. Add input/output `Stream`s with lab-scale `mass_per_batch_g` (the first
   input is the limiting feed).
2. Declare `ProcessSection`s (PFD blocks) and optional explicit `edges`.
3. Add `Equipment` items with installed cost in CEPCI_ref dollars at
   `cap_ref` ton/batch and a `scaling_factor` (typical 0.6).
4. Optionally add per-ton/y utility coefficients in `process.meta`
   under keys ending in `_$_per_ton_per_y`.
5. Optionally add `process.extra_opex` (e.g. distillation OPEX) and
   `process.extra_capex_annualized` (initial inventory, distillation
   column, etc.) - both linear in ton.

## What the xlsx export contains

Sheet **TEA Summary** (mirrors the reference workbook):

1. Experimental Conditions & Scale-up
2. Key Assumptions
3. CAPEX Summary (per section, per scale)
4. OPEX Summary (feedstock / utility / operation totals)
5. Revenue per product
6. Profitability (CAPEX + OPEX vs Revenue)
7. MSP for the chosen product, per scale
8. Cost & revenue breakdown at the largest scale
9. Sensitivity sweeps (if any were run in the UI)

Sheet **Process Detail**: full stream and equipment tables.
